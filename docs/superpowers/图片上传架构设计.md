# 图片上传架构设计说明

## 🎯 真实场景的图片上传方案

### 核心原则
在真实的生产环境中，**前端直接上传到OSS**是最佳实践，原因如下：

| 对比项 | 前端直接上传OSS | 后端代理上传 |
|--------|-----------------|--------------|
| **服务器带宽** | ✅ 不占用 | ❌ 占用大量带宽 |
| **服务器压力** | ✅ 减轻压力 | ❌ 增加CPU和内存压力 |
| **上传速度** | ✅ 更快（直连OSS） | ⚠️ 较慢（经转发） |
| **大文件支持** | ✅ 支持断点续传 | ⚠️ 容易超时 |
| **成本** | ✅ 降低服务器成本 | ❌ 增加服务器成本 |
| **控制力** | ⚠️ 需要STS凭证管理 | ✅ 完全可控 |

## 🔐 安全设计

### 1. STS临时凭证机制

```java
/**
 * STS凭证生成流程
 */
public class StsService {
    public StsToken generateUploadToken(Long userId, String fileName) {
        // 1. 创建AssumeRole请求
        AssumeRoleRequest request = new AssumeRoleRequest();
        request.setRoleArn("acs:ram::ACCOUNT_ID:role/OssUploadRole");
        request.setRoleSessionName("auction-upload-" + userId);
        request.setDurationSeconds(3600); // 1小时有效期

        // 2. 添加权限限制（可选）
        request.setPolicy(generateUploadPolicy(userId));

        // 3. 获取临时凭证
        AssumeRoleResponse response = stsClient.getAcsResponse(request);

        // 4. 返回凭证
        return new StsToken(
            response.getCredentials().getAccessKeyId(),
            response.Credentials.getAccessKeySecret(),
            response.getCredentials().getSecurityToken(),
            response.Credentials().getExpiration()
        );
    }
}
```

### 2. 权限策略限制

**限制用户只能上传到指定路径**：
```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:PutObject"],
      "Resource": ["acs:oss:*:*:products/USER_ID/*"]
    }
  ]
}
```

**安全优势**：
- ✅ 凭证只有1小时有效期
- ✅ 只能上传到指定路径
- ✅ 只能执行PutObject操作
- ✅ 无法读取其他文件

### 3. 文件验证

**后端验证（OssStorageService）**：
```java
private void validateFile(MultipartFile file) {
    // 1. 检查文件大小
    if (file.getSize() > MAX_FILE_SIZE) {
        throw new IllegalArgumentException("文件过大");
    }

    // 2. 检查文件类型
    String extension = getFileExtension(file.getOriginalFilename());
    if (!isAllowedExtension(extension)) {
        throw new IllegalArgumentException("不支持的文件格式");
    }

    // 3. 检查Content-Type
    if (!file.getContentType().startsWith("image/")) {
        throw new IllegalArgumentException("只允许图片");
    }
}
```

**前端验证（推荐双重验证）**：
```javascript
// 前端也应该验证文件
function validateFile(file) {
    // 1. 文件大小
    if (file.size > 10 * 1024 * 1024) {
        alert('文件不能超过10MB');
        return false;
    }

    // 2. 文件类型
    if (!file.type.startsWith('image/')) {
        alert('只能上传图片');
        return false;
    }

    return true;
}
```

## 📊 数据流详解

### 场景1：创建商品（包含图片上传）

```
用户操作：点击"创建商品"按钮
    ↓
前端：显示表单，包含图片上传组件
    ↓
用户操作：选择图片文件
    ↓
前端：验证文件（大小、类型）
    ↓
前端：请求后端获取上传凭证
    POST /upload/token
    Headers: { Authorization: "Bearer JWT_TOKEN" }
    ↓
后端：验证JWT，解析userId
    ↓
后端：调用STS服务生成临时凭证
    ↓
后端：返回STS凭证（包含accessKeyId、accessKeySecret、stsToken）
    ↓
前端：使用阿里云OSS SDK初始化客户端
    ↓
前端：直接上传图片到OSS
    PUT https://bucket.oss-cn-hangzhou.aliyuncs.com/products/2026/06/07/uuid.jpg
    Headers: { Authorization: "OSS stsToken" }
    ↓
OSS：验证STS凭证
    ↓
OSS：保存文件
    ↓
OSS：返回图片URL
    https://bucket.oss-cn-hangzhou.aliyuncs.com/products/2026/06/07/uuid.jpg
    ↓
前端：接收URL，显示预览
    ↓
用户操作：填写其他商品信息，点击"提交"
    ↓

    POST /products
```

## 🔄 备用方案：后端代理上传

### 使用场景
- 小文件（< 1MB）
- 需要后端处理的文件（水印、压缩等）
- 特殊环境（不支持前端直接上传）

### 实现方式
```java
/**
 * 后端代理上传
 */
@PostMapping("/upload/image")
public Result<ImageUploadResponse> uploadImage(
        @RequestParam("file") MultipartFile file,
        @CurrentUser UserPrincipal currentUser) {

    // 1. 验证文件
    validateFile(file);

    // 2. 可选：图片处理
    // Image processedImage = imageProcessor.compress(file);

    // 3. 上传到OSS
    String imageUrl = ossStorageService.uploadFile(file, file.getOriginalFilename());

    // 4. 记录上传日志
    log.info("图片上传成功: userId={}, url={}", currentUser.getUserId(), imageUrl);

    // 5. 返回URL
    return Result.ok(new ImageUploadResponse(imageUrl));
}
```

## 🚀 性能优化

### 1. CDN加速
```
用户请求图片 → CDN边缘节点 → 缓存命中 → 快速返回
                    ↓ 缓存未命中
                    OSS源站 → 返回图片 → CDN缓存
```

### 2. 图片压缩
```javascript
// 前端压缩
function compressImage(file, quality = 0.8) {
    return new Promise((resolve) => {
        const reader = new FileReader();
        reader.onload = (e) => {
            const img = new Image();
            img.onload = () => {
                const canvas = document.createElement('canvas');
                // 压缩逻辑...
                canvas.toBlob(resolve, 'image/jpeg', quality);
            };
            img.src = e.target.result;
        };
        reader.readAsDataURL(file);
    });
}
```

### 3. 分片上传
```javascript
// 大文件分片上传
async function multipartUpload(file) {
    const result = await client.multipartUpload(
        'products/large-file.jpg',
        file,
        {
            parallel: 3,      // 并发上传3片
            partSize: 1024 * 1024,  // 每片1MB
            progress: (p) => console.log('进度:', p)
        }
    );
}
```

## 🔧 环境准备

### 1. 阿里云OSS配置
```bash
# 1. 创建OSS Bucket
- 名称：auction-products
- 区域：oss-cn-hangzhou
- 权限：私有读写

# 2. 配置CORS
- 允许的来源：*
- 允许的方法：GET, POST, PUT, DELETE
- 允许的头部：*
- 暴露的头部：ETag

# 3. 创建RAM角色
- 角色名称：OssUploadRole
- 信任策略：允许当前账号扮演此角色

# 4. 配置角色权限
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "oss:PutObject"
      ],
      "Resource": [
        "acs:oss:*:*:auction-products/*"
      ]
    }
  ]
}
```

### 2. 应用配置
```yaml
# application.yml
oss:
  region: oss-cn-hangzhou
  endpoint: https://oss-cn-hangzhou.aliyuncs.com
  access-key-id: YOUR_ACCESS_KEY_ID
  access-key-secret: YOUR_ACCESS_KEY_SECRET
  bucket-name: auction-products
  key-prefix: products/
  cdn-domain: https://cdn.yourdomain.com

  token-expire-time: 3600
  allowed-extensions: jpg,jpeg,png,gif,webp
```

## 🎯 总结

### 推荐方案：前端直接上传OSS
- ✅ 减轻服务器压力
- ✅ 提升上传速度
- ✅ 支持大文件和断点续传
- ✅ 降低服务器成本

### 关键技术点
1. **STS临时凭证**：安全地提供上传权限
2. **前端OSS SDK**：直接与OSS通信
3. **权限策略**：限制上传路径和操作
4. **文件验证**：前后端双重验证
5. **CDN加速**：提升图片访问速度

这个方案完全符合真实生产环境的最佳实践！
