# 商品管理模块 - 防御性编程实现说明

## 📋 概述

本模块实现了完整的商品管理功能，采用防御性编程设计，确保系统的健壮性和安全性。

---

## 🛡️ 防御性编程特性

### 1. 参数验证层

#### 请求体验证（自动）
- 使用 `@Valid` 注解自动验证DTO
- 使用 `@NotNull`, `@NotBlank`, `@Size` 等标准验证注解
- 自定义验证规则（价格范围、库存数量等）

#### 业务逻辑验证（手动）
```java
// ID有效性验证
DefensiveCheck.validId(id, "商品ID");

// 对象非空验证
DefensiveCheck.notNull(obj, "字段名");

// 资源存在性验证
DefensiveCheck.exists(product, "商品");
```

---

### 2. 权限控制层

#### 角色验证
- 普通用户：只能查看已上架商品
- 商家：可以创建、修改自己的商品
- 管理员：可以管理所有商品

#### 资源所有权验证
```java
// 验证商品所有权（商家只能操作自己的商品）
DefensiveCheck.ownership(product.getMerchantId(), currentUserId, "商品");
```

---

### 3. 业务规则验证

#### SKU唯一性检查
```java
if (productRepository.existsBySku(sku)) {
    throw new BizException(ErrorCode.BAD_REQUEST, "商品编码已存在");
}
```

#### 状态转换验证
```java
// 验证状态转换合法性
DefensiveCheck.statusTransition(currentStatus, newStatus);

// 状态转换规则：
// PENDING_REVIEW → LISTED, DELISTED
// LISTED → DELISTED
// DELISTED → LISTED
```

#### 数值范围验证
```java
// 价格范围：0 < price <= 99999999
// 库存范围：0 <= stock <= 999999
// 字符串长度限制
DefensiveCheck.length(str, maxLen, "字段名");
```

---

### 4. 安全防护

#### SQL注入防护
```java
// 检查常见的SQL注入关键词
String sqlKeywords = {"'", "\"", ";", "--", "/*", "*/", "xp_", "exec", "execute"};
```

#### XSS防护
```java
// 验证图片URL安全性
if (imageUrl.contains("<") || imageUrl.contains(">") ||
    imageUrl.contains("javascript:") || imageUrl.contains("data:")) {
    throw new BizException(ErrorCode.BAD_REQUEST, "图片URL不安全");
}
```

---

### 5. 并发控制

#### 乐观锁机制
```java
// 在Product实体中包含version字段
@TableVersion
private Integer version;

// 更新时自动检查版本号
int rows = productMapper.updateById(product);
if (rows == 0) {
    throw new BizException(ErrorCode.CONFLICT, "数据已被修改，请刷新后重试");
}
```

### 7. 审计日志

## 🔗 接口清单

| 接口 | 方法 | 路径 | 功能 | 权限 |
|------|------|------|------|------|
| 创建商品 | POST | /products | 创建商品基本信息 | 商家 |
| 查询商品详情 | GET | /products/{id} | 获取完整商品信息 | 所有人 |
| 查询商品列表 | GET | /products | 分页查询、筛选、搜索 | 所有人 |
| 更新商品 | PUT | /products/{id} | 更新商品信息 | 商家（仅自己的） |
| 上架商品 | POST | /products/{id}/list | 上架商品 | 商家（仅自己的） |
| 下架商品 | POST | /products/{id}/delist | 下架商品 | 商家（仅自己的） |
| 上传图片 | POST | /products/{id}/images | 上传商品图片 | 商家（仅自己的） |
| 删除图片 | DELETE | /products/{id}/images/{imageId} | 删除商品图片 | 商家（仅自己的） |

## 📝 请求示例

### 1. 创建商品
```bash
POST /products
Headers:
  X-User-Id: 1
  X-User-Role: MERCHANT

Request Body:
{
  "merchantId": 1,
  "sku": "IPHONE15PRO",
  "name": "iPhone 15 Pro Max",
  "brand": "Apple",
  "imageUrl": "https://example.com/iphone.jpg",
  "description": "全新iPhone 15 Pro Max",
  "category": "手机数码",
  "initialPrice": 7999.00,
  "stock": 10
}
```

### 2. 查询商品列表
```bash
GET /products?page=1&size=20&category=手机数码&status=LISTED&sortBy=initialPrice&sortOrder=asc
Headers:
  X-User-Id: 2
  X-User-Role: USER
```

### 3. 更新商品
```bash
PUT /products/1
Headers:
  X-User-Id: 1
  X-User-Role: MERCHANT

Request Body:
{
  "name": "iPhone 15 Pro Max 256G",
  "initialPrice": 8999.00
}
```

### 4. 上架商品
```bash
POST /products/1/list
Headers:
  X-User-Id: 1
  X-User-Role: MERCHANT
```

### 5. 上传图片
```bash
POST /products/1/images
Headers:
  X-User-Id: 1
  X-User-Role: MERCHANT

Request Body:
{
  "imageUrl": "https://example.com/new-image.jpg",
  "isPrimary": true
}
```
