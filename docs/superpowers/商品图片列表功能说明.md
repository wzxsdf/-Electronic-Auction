# 商品图片列表功能说明

## 功能概述

商品支持上传多张图片（最多4张），前端传来的图片列表会以**逗号分隔的字符串**形式存储到数据库。

### 存储格式

**前端传来：**
```json
{
  "imageUrls": [
    "https://example.com/image1.jpg",
    "https://example.com/image2.jpg",
    "https://example.com/image3.jpg",
    "https://example.com/image4.jpg"
  ]
}
```

**数据库存储：**
```
image_url (varchar)
https://example.com/image1.jpg,https://example.com/image2.jpg,https://example.com/image3.jpg,https://example.com/image4.jpg
```

## API 接口

### 创建商品（支持图片列表）

**接口：** `POST /products`

**请求头：**
```
Authorization: Bearer <your_jwt_token>
Content-Type: application/json
```

**请求体：**
```json
{
  "name": "Apple iPhone 15 Pro Max",
  "brand": "Apple",
  "imageUrls": [
    "https://example.com/images/iphone-front.jpg",
    "https://example.com/images/iphone-side.jpg",
    "https://example.com/images/iphone-back.jpg"
  ],
  "description": "全新iPhone 15 Pro Max，256GB",
  "category": "手机数码",
  "initialPrice": 9999.00,
  "stock": 10
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "merchantId": 2,
    "name": "Apple iPhone 15 Pro Max",
    "brand": "Apple",
    "imageUrl": "https://example.com/images/iphone-front.jpg,https://example.com/images/iphone-side.jpg,https://example.com/images/iphone-back.jpg",
    "imageUrls": [
      "https://example.com/images/iphone-front.jpg",
      "https://example.com/images/iphone-side.jpg",
      "https://example.com/images/iphone-back.jpg"
    ],
    "primaryImageUrl": "https://example.com/images/iphone-front.jpg",
    "description": "全新iPhone 15 Pro Max，256GB",
    "category": "手机数码",
    "initialPrice": 9999.00,
    "stock": 10,
    "status": "PENDING_REVIEW",
    "createdAt": "2024-06-08T10:30:00"
  }
}
```

## 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| imageUrl | String | 图片URL字符串（逗号分隔），向后兼容 |
| imageUrls | List[String] | 图片URL列表，前端建议使用此字段 |
| primaryImageUrl | String | 主图URL（第一张图片） |

## 业务规则

### 图片数量限制
- **最多4张图片**
- 超过限制会返回错误：`最多只能上传4张图片`

### 图片URL验证
- 必须以 `http://` 或 `https://` 开头
- 单个URL最大长度500字符
- 不允许包含危险字符：`<`、`>`
- 不允许使用危险协议：`javascript:`、`data:`

### 主图规则
- **第一张图片为主图**
- 通过 `primaryImageUrl` 字段可以直接获取主图

## 错误处理

### 图片数量超限
```json
{
  "code": 400,
  "message": "最多只能上传4张图片"
}
```

### 图片URL不安全
```json
{
  "code": 400,
  "message": "图片URL不安全: javascript:alert(1)"
}
```

### 图片格式错误
```json
{
  "code": 400,
  "message": "图片URL格式不正确，必须以http://或https://开头: ftp://example.com/image.jpg"
}
```

## 数据示例

### 数据库中的实际存储

```sql
SELECT id, name, image_url FROM products WHERE id = 1;
```

**结果：**
```
id | name                      | image_url
---+---------------------------+--------------------------------------------------
1  | Apple iPhone 15 Pro Max  | https://example.com/img1.jpg,https://example.com/img2.jpg
```

### 查询商品时的解析

```java
Product product = productRepository.findById(1);

// 获取原始字符串
String imageUrl = product.getImageUrl();
// "https://example.com/img1.jpg,https://example.com/img2.jpg"

// 获取图片列表
List<String> imageUrls = product.getImageList();
// ["https://example.com/img1.jpg", "https://example.com/img2.jpg"]

// 获取主图
String primaryImageUrl = product.getPrimaryImageUrl();
// "https://example.com/img1.jpg"
```

## 前端使用建议

### 1. 图片上传组件
```javascript
// 前端图片上传
const uploadImages = async (files) => {
  const urls = [];
  for (let file of files) {
    const url = await uploadToOSS(file); // 上传到OSS
    urls.push(url);
  }
  return urls;
};

// 创建商品
const createProduct = async (productData) => {
  const response = await axios.post('/products', {
    ...productData,
    imageUrls: productData.images // 传递图片URL列表
  });
  return response.data;
};
```

### 2. 图片展示
```javascript
// 显示所有图片
const ProductGallery = ({ product }) => {
  return (
    <div className="image-gallery">
      {product.imageUrls.map((url, index) => (
        <img key={index} src={url} alt={product.name} />
      ))}
    </div>
  );
};

// 显示主图
const ProductCard = ({ product }) => {
  return (
    <div className="product-card">
      <img src={product.primaryImageUrl} alt={product.name} />
      <h3>{product.name}</h3>
    </div>
  );
};
```

## 优势与特点

### ✅ 简单高效
- 无需额外的数据库表
- 一个字段存储所有图片
- 查询性能好

### ✅ 易于使用
- 前端直接传列表
- 后端自动转换
- 向后兼容（保留imageUrl字段）

### ✅ 灵活扩展
- 限制4张图片，满足大多数场景
- 可以轻松调整限制数量
- 支持未来添加更多图片属性

### ✅ 数据完整
- 自动去重
- 自动过滤空值
- 自动验证安全性

## 注意事项

1. **图片存储**：系统只保存图片URL，不实际存储图片文件
2. **URL有效期**：请确保提供的图片URL长期有效
3. **图片大小**：建议单张图片不超过2MB
4. **图片格式**：推荐使用 JPG、PNG、WebP 格式
5. **主图建议**：第一张图片作为主图，建议使用高质量图片
6. **字符限制**：总URL字符串长度受数据库字段限制（varchar 500）

## 扩展建议

如果未来需要更复杂的图片管理功能，可以考虑：
- 添加图片标题/描述字段
- 添加图片排序字段
- 添加图片分类（正面图、侧面图等）
- 独立的图片表（参考之前回滚的方案）

但对于大多数电商场景，当前方案已经足够使用。
