模块 B  @不得不熬夜
本周进展
后端核心业务功能开发：
1. 竞拍核心业务逻辑完整实现
- 出价服务（BidService）完成：包含固定加价幅度校验、封顶价校验、分布式锁防并发
- Redis实时缓存当前价格和最高出价者，支持高并发场景
- 竞拍状态管理：实现PENDING→ACTIVE→COMPLETED/CANCELLED状态流转
- 出价记录完整保存，支持排行榜计算和参与统计
2. 商品与竞拍管理功能
- ProductController：商品发布、查询所有商品、根据ID查询
- AuctionController：创建竞拍、查询竞拍列表、竞拍详情查询、取消竞拍
- 支持竞拍规则配置：起拍价、加价幅度、竞拍时长、封顶价、自动延时时间
3. WebSocket实时推送架构
- WsMessageService：新出价广播、价格更新广播、领先/被超越通知
- AuctionWebSocketHandler：支持用户加入/退出直播间
- 异步消息推送（@Async），提升高并发下的响应速度
4. 并发控制与性能优化
- Redisson分布式锁：防止同一竞拍的并发出价冲突
- 限流功能：出价接口30次/分钟限流，防止恶意刷单
- 并发测试验证：ConcurrentBidTest验证100线程并发出价正确性
  识别到问题/风险
1. 核心业务闭环缺失：竞拍结束定时任务、自动结算逻辑、订单生成、模拟支付功能未实现
2. 定时任务框架未启用：@Scheduled注解未配置，无法实现自动延时和自动结束
3. Redis与数据库数据一致性：双写场景下可能存在不一致风险，缺少同步机制
4. 权限控制完全缺失：用户角色未区分，无法保证数据安全
