## 1. 系统概述

### 1.1 功能简介

订单管理模块负责拍卖成交后的订单生成、状态管理和支付流程，是连接拍卖与支付的桥梁，包括：

- ✅ **自动订单生成**: 拍卖成交后自动为买家创建订单
- ✅ **订单状态管理**: 完整的订单生命周期状态流转
- ✅ **订单查询**: 用户和商家的订单查询功能
- ✅ **支付集成**: 与支付模块的集成接口
- ✅ **订单统计**: 用户订单数据统计分析
- ✅ **异常处理**: 订单异常情况的处理机制

### 1.2 设计目标

- **自动化**: 拍卖成交后自动生成订单，无需人工干预
- **一致性**: 保证订单状态与支付状态的一致性
- **可追溯**: 完整的订单历史记录和状态变更记录
- **安全性**: 确保订单数据的准确性和安全性
- **扩展性**: 支持多种订单类型和支付方式

### 1.3 技术特点

| 特性 | 技术方案 | 优势 |
|------|----------|------|
| 状态机 | 严格状态流转控制 | 保证业务逻辑正确 |
| 自动生成 | 事件驱动机制 | 无需人工干预 |
| 数据关联 | 多表关联查询 | 提供完整订单信息 |
| 事务管理 | Spring事务管理 | 保证数据一致性 |
| 异步处理 | 支付异步处理 | 提升用户体验 |

### 2.2 订单状态机

```
┌─────────────────────────────────────────────────────────┐
│                    订单状态流转图                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │  PENDING_PAYMENT → PAID → COMPLETED              │   │
│  │       ↓                                      ↑    │   │
│  │  CANCELLED                                   │    │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘

状态说明：
- PENDING_PAYMENT: 待支付 (初始状态)
- PAID: 已支付 (支付完成)
- COMPLETED: 已完成 (确认收货)
- CANCELLED: 已取消 (超时或用户取消)
```

**字段说明**:
- `id`: 主键，自增长
- `order_no`: 订单号，全局唯一
- `user_id`: 买家用户ID
- `final_amount`: 拍卖成交金额
- `status`: 订单状态（PENDING_PAYMENT, PAID, COMPLETED, CANCELLED）
- `paid_time`: 支付时间
- `cancel_reason`: 取消原因

## 4. 核心功能

### 4.1 订单创建

```java
/**
 * 创建订单（由结算服务调用）
 */
@Transactional(rollbackFor = Exception.class)
public Order createOrder(Order order) {
    // 1. 验证必要字段
    validateOrder(order);
    
    // 2. 生成订单号
    order.setOrderNo(generateOrderNo());
    
    // 3. 设置初始状态
    if (order.getStatusEnum() == null) {
        order.setStatusEnum(OrderStatus.PENDING_PAYMENT);
    }
    
    // 4. 保存订单
    Order savedOrder = orderRepository.save(order);
    
    // 5. 记录状态变更日志
    recordStatusChange(savedOrder.getId(), null, OrderStatus.PENDING_PAYMENT, 
                      "SYSTEM", "订单创建");
    
    // 6. 发送订单创建通知
    sendOrderCreatedNotification(savedOrder);
    
    log.info("订单创建成功: orderId={}, orderNo={}, userId={}, amount={}",
            savedOrder.getId(), savedOrder.getOrderNo(), 
            order.getUserId(), order.getFinalAmount());
    
    return savedOrder;
}

/**
 * 生成订单号
 */
private String generateOrderNo() {
    // 格式: AU + 年月日时分秒 + 6位随机数
    String datetime = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
    String random = String.format("%06d", new Random().nextInt(1000000));
    return "AU" + datetime + random;
}
```

```java
public Order getOrderDetail(Long orderId) {
    // 1. 查询订单基本信息
    Order order = orderRepository.findById(orderId);
    if (order == null) {
        throw new BizException(ErrorCode.ORDER_NOT_FOUND);
    }
    
    // 2. 加载关联信息
    enrichOrderInfo(order);
    
    return order;
}

/**
 * 为订单加载关联信息
 */
private void enrichOrderInfo(Order order) {
    // 加载拍卖信息
    if (order.getItemId() != null) {
        Auction auction = auctionRepository.findById(order.getItemId());
        if (auction != null) {
            order.setAuctionTitle(auction.getTitle());
            order.setAuctionStartTime(auction.getStartTime());
            order.setAuctionEndTime(auction.getEndTime());
        }
    }
    
    // 加载商品信息
    if (order.getProductId() != null) {
        Product product = productRepository.findById(order.getProductId());
        if (product != null) {
            order.setProductName(product.getName());
            order.setProductImageUrl(product.getImageUrl());
            order.setProductDescription(product.getDescription());
        }
    }
    
    // 加载用户信息
    if (order.getUserId() != null) {
        User user = userRepository.findById(order.getUserId());
        if (user != null) {
            order.setBuyerNickname(user.getNickname());
            order.setBuyerPhone(user.getPhone());
        }
    }
}

public List<Order> getUserOrders(Long userId) {
```

### 4.3 订单状态管理

```java
/**
 * 更新订单状态
 */
@Transactional(rollbackFor = Exception.class)
public void updateOrderStatus(Long orderId, OrderStatus newStatus) {
    // 1. 查询当前订单
    Order order = orderRepository.findById(orderId);
    if (order == null) {
        throw new BizException(ErrorCode.ORDER_NOT_FOUND);
    }
    
    OrderStatus oldStatus = order.getStatusEnum();
    
    // 2. 验证状态转换是否合法
    validateStatusTransition(oldStatus, newStatus);
    
    // 3. 更新订单状态
    order.setStatusEnum(newStatus);
    
    // 4. 根据状态更新时间字段
    switch (newStatus) {
        case PAID:
            order.setPaidTime(LocalDateTime.now());
            break;
        case COMPLETED:
            order.setCompletedTime(LocalDateTime.now());
            break;
        case CANCELLED:
            order.setCancelledTime(LocalDateTime.now());
            break;
        default:
            break;
    }
    
    // 5. 保存订单
    orderRepository.save(order);
    
    // 6. 记录状态变更日志
    recordStatusChange(orderId, oldStatus, newStatus, "SYSTEM", "状态自动更新");
    
    log.info("订单状态更新: orderId={}, oldStatus={}, newStatus={}",
            orderId, oldStatus, newStatus);
}

/**
 * 验证订单状态转换是否合法
 */
private void validateStatusTransition(OrderStatus currentStatus, OrderStatus newStatus) {
    switch (currentStatus) {
        case PENDING_PAYMENT:
            if (newStatus != OrderStatus.PAID && newStatus != OrderStatus.CANCELLED) {
                throw new BizException(ErrorCode.ORDER_STATUS_INVALID,
                    "待支付订单只能变更为已支付或已取消状态");
            }
            break;
        case PAID:
            if (newStatus != OrderStatus.COMPLETED) {
                throw new BizException(ErrorCode.ORDER_STATUS_INVALID,
                    "已支付订单只能变更为已完成状态");
            }
            break;
        case COMPLETED:
        case CANCELLED:
            throw new BizException(ErrorCode.ORDER_STATUS_INVALID,
                "已完成或已取消的订单状态不可变更");
        default:
            break;
    }
}

/**
 * 取消订单
 */
@Transactional(rollbackFor = Exception.class)
public void cancelOrder(Long orderId, String reason) {
    Order order = orderRepository.findById(orderId);
    if (order == null) {
        throw new BizException(ErrorCode.ORDER_NOT_FOUND);
    }
    
    // 只有待支付状态的订单才能取消
    if (order.getStatusEnum() != OrderStatus.PENDING_PAYMENT) {
        throw new BizException(ErrorCode.ORDER_STATUS_INVALID,
            "当前订单状态不允许取消");
    }
    
    // 更新订单状态
    order.setStatusEnum(OrderStatus.CANCELLED);
    order.setCancelReason(reason);
    order.setCancelledTime(LocalDateTime.now());
    orderRepository.save(order);
    
    // 记录状态变更日志
    recordStatusChange(orderId, OrderStatus.PENDING_PAYMENT, OrderStatus.CANCELLED,
                      "USER", reason);
    
    log.info("订单取消成功: orderId={}, reason={}", orderId, reason);
}
```

## 6. 业务规则

### 6.1 订单创建规则

- **自动创建**: 拍卖成交后系统自动创建订单
- **买家确认**: 最高出价者自动成为买家
- **金额确定**: 订单金额等于拍卖成交价格
- **初始状态**: 订单创建后默认为待支付状态
- **支付时效**: 订单创建后需在规定时间内完成支付

### 6.2 订单状态规则

| 当前状态 | 允许转换的状态 | 说明 |
|----------|----------------|------|
| PENDING_PAYMENT | PAID, CANCELLED | 待支付订单可支付或取消 |
| PAID | COMPLETED | 已支付订单可完成 |
| COMPLETED | - | 已完成订单不可变更 |
| CANCELLED | - | 已取消订单不可变更 |

### 6.3 订单查询权限

- **用户查询**: 只能查询自己的订单
- **商家查询**: 可查询自己商品的订单
- **管理员查询**: 可查询所有订单

```java
        if (paymentStatus != PaymentStatus.SUCCESS) {
            log.warn("订单状态与支付状态不一致: orderId={}, orderStatus={}, paymentStatus={}",
                    order.getId(), order.getStatus(), paymentStatus);
        }
    });
}
```

## 10. 常见问题

### Q1: 订单号是如何生成的？

**A**: 订单号格式为 `AU + 年月日时分秒 + 6位随机数`，确保全局唯一性。

### Q2: 订单超时未支付如何处理？

**A**: 系统会定时检查待支付订单的创建时间，超时未支付的订单会自动取消。

### Q3: 用户可以取消已支付的订单吗？

**A**: 不可以，只有待支付状态的订单可以取消，已支付订单需要走退款流程。

### Q4: 如何查询订单的完整信息？

**A**: 调用订单详情查询接口，系统会自动加载关联的拍卖、商品、用户等信息。
