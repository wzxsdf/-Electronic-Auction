## 1. 系统概述

### 1.1 功能简介

支付管理模块负责处理订单支付流程，提供模拟支付功能用于测试和演示，包括：

- ✅ **模拟支付**: 提供模拟支付功能，支持异步处理
- ✅ **支付状态管理**: 完整的支付状态跟踪
- ✅ **支付结果查询**: 支付结果和状态查询
- ✅ **支付通知**: 支付成功/失败的实时通知
- ✅ **异常处理**: 支付异常情况的处理
- ✅ **统计分析**: 支付数据统计分析

### 1.2 设计目标

- **真实性**: 模拟真实支付流程的各个环节
- **异步性**: 支持异步支付处理，提升用户体验
- **可靠性**: 保证支付结果的一致性和准确性
- **可测试性**: 提供可控的测试环境
- **扩展性**: 支持接入真实支付渠道

### 1.3 技术特点

| 特性 | 技术方案 | 优势 |
|------|----------|------|
| 异步处理 | CompletableFuture | 提升响应速度 |
| 随机模拟 | 随机延迟和失败率 | 模拟真实场景 |
| 状态管理 | 严格状态流转 | 保证数据一致性 |
| 实时通知 | WebSocket推送 | 即时状态更新 |
| 线程池 | 专用线程池 | 资源可控 |

## 3. 数据库设计

### 3.1 支付记录表 (payment_records)

```sql
CREATE TABLE `payment_records` (
  `id`                BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '支付记录ID',
  `order_id`          BIGINT NOT NULL COMMENT '订单ID',
  `order_no`          VARCHAR(50) NOT NULL COMMENT '订单号',
  `user_id`           BIGINT NOT NULL COMMENT '支付用户ID',
  `amount`            DECIMAL(10,2) NOT NULL COMMENT '支付金额',
  `status`            VARCHAR(20) NOT NULL COMMENT '支付状态',
  `payment_method`    VARCHAR(50) COMMENT '支付方式',
  `transaction_id`    VARCHAR(100) COMMENT '交易流水号',
  `success_time`      DATETIME COMMENT '支付成功时间',
  `failure_reason`    VARCHAR(500) COMMENT '失败原因',
  `ip_address`        VARCHAR(50) COMMENT 'IP地址',
  `created_at`        DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at`        DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  INDEX `idx_order_id` (`order_id`),
  INDEX `idx_order_no` (`order_no`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_created_at` (`created_at`),
  FOREIGN KEY (`order_id`) REFERENCES `orders`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='支付记录表';
```

**字段说明**:
- `id`: 主键，自增长
- `order_id`: 关联的订单ID
- `status`: 支付状态（PENDING, SUCCESS, FAILED, CANCELLED）
- `payment_method`: 支付方式（ALIPAY, WECHAT, MOCK等）
- `transaction_id`: 第三方支付平台的交易流水号
- `success_time`: 支付成功时间

## 4. 核心功能

### 4.1 模拟支付处理

```java
/**
 * 异步模拟支付
 */
@Async("paymentTaskExecutor")
public CompletableFuture<PaymentResult> processPayment(Long orderId, Long userId) {
    log.info("开始处理支付: orderId={}, userId={}", orderId, userId);
    
    return CompletableFuture.supplyAsync(() -> {
        try {
            // 1. 验证订单
            Order order = validateOrderForPayment(orderId, userId);
            
            // 2. 模拟支付处理延迟（1-3秒）
            simulatePaymentDelay();
            
            // 3. 模拟支付处理（随机失败，10%失败率）
            boolean success = simulatePaymentProcessing(order);
            
            if (success) {
                // 4. 支付成功，更新订单状态
                completePayment(order);
                log.info("支付处理成功: orderId={}, userId={}", orderId, userId);
            } else {
                // 5. 支付失败
                log.warn("支付处理失败: orderId={}, userId={}", orderId, userId);
                throw new BizException(ErrorCode.PAYMENT_FAILED, "支付处理失败，请重试");
            }
            
            return PaymentResult.success(orderId, order.getFinalAmount());
            
        } catch (BizException e) {
            log.error("支付处理失败: orderId={}, error={}", orderId, e.getMessage());
            return PaymentResult.failure(orderId, e.getMessage());
        } catch (Exception e) {
            log.error("支付处理异常: orderId={}", orderId, e);
            return PaymentResult.failure(orderId, "支付处理异常");
        }
    });
}

/**
 * 模拟支付处理延迟（1-3秒）
 */
private void simulatePaymentDelay() {
    try {
        int delayMs = 1000 + random.nextInt(2000); // 1-3秒随机延迟
        TimeUnit.MILLISECONDS.sleep(delayMs);
        log.debug("模拟支付延迟: {}ms", delayMs);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        log.warn("支付延迟被中断");
    }
}

/**
 * 模拟支付处理逻辑
 */
private boolean simulatePaymentProcessing(Order order) {
    // 90%成功率，10%失败率用于测试异常场景
    boolean success = random.nextDouble() > 0.1;
    
    if (success) {
        log.info("支付处理模拟成功: orderId={}, amount={}",
                order.getId(), order.getFinalAmount());
    } else {
        log.warn("支付处理模拟失败: orderId={}, amount={}",
                order.getId(), order.getFinalAmount());
    }
    
    return success;
}
```

### 4.2 支付完成处理

```java
/**
 * 完成支付，更新订单状态并发送通知
 */
private void completePayment(Order order) {
    // 1. 更新订单状态为已支付
    order.setStatusEnum(OrderStatus.PAID);
    orderRepository.save(order);
    
    // 2. 创建支付记录
    PaymentRecord record = new PaymentRecord();
    record.setOrderId(order.getId());
    record.setOrderNo(order.getOrderNo());
    record.setUserId(order.getUserId());
    record.setAmount(order.getFinalAmount());
    record.setStatus("SUCCESS");
    record.setPaymentMethod("MOCK");
    record.setTransactionId(generateTransactionId());
    record.setSuccessTime(LocalDateTime.now());
    paymentRecordRepository.save(record);
    
    // 3. 发送WebSocket通知
    wsMessageService.sendPaymentSuccess(
        order.getUserId(), 
        order.getId(),
        order.getFinalAmount()
    );
    
    // 4. 记录支付日志
    log.info("订单支付完成: orderId={}, userId={}, amount={}, paidTime={}",
            order.getId(), order.getUserId(), 
            order.getFinalAmount(), LocalDateTime.now());
}

/**
 * 生成交易流水号
 */
private String generateTransactionId() {
    // 格式: MOCK + 年月日时分秒 + 6位随机数
    String datetime = LocalDateTime.now()
        .format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
    String random = String.format("%06d", new Random().nextInt(1000000));
    return "MOCK" + datetime + random;
}
```

        log.error("同步支付失败: orderId={}, error={}", orderId, e.getMessage());
        return PaymentResult.failure(orderId, e.getMessage());

### 4.4 支付状态查询

```java
/**
 * 查询支付状态
 */
public PaymentStatus getPaymentStatus(Long orderId) {
    Order order = orderRepository.findById(orderId);
    if (order == null) {
        throw new BizException(ErrorCode.ORDER_NOT_FOUND);
    }
    
    return switch (order.getStatusEnum()) {
        case PENDING_PAYMENT -> PaymentStatus.PENDING;
        case PAID, COMPLETED -> PaymentStatus.SUCCESS;
        case CANCELLED -> PaymentStatus.CANCELLED;
        default -> PaymentStatus.UNKNOWN;
    };
}
```

## 6. 业务规则

### 6.1 支付验证规则

| 验证项 | 验证条件 | 错误提示 |
|--------|----------|----------|
| 订单存在 | 订单ID必须存在 | 订单不存在 |
| 订单所属 | 订单必须属于当前用户 | 无权操作此订单 |
| 订单状态 | 订单状态必须为待支付 | 订单状态不允许支付 |
| 重复支付 | 订单不能重复支付 | 订单已支付 |
| 金额验证 | 支付金额必须正确 | 支付金额不匹配 |

### 6.2 支付处理规则

- **异步处理**: 默认使用异步处理，1-3秒延迟
- **成功率**: 模拟90%成功率，10%失败率
- **重试机制**: 失败后可重新发起支付
- **时效性**: 支付请求需在订单创建后24小时内完成

### 6.3 支付通知规则

| 通知类型 | 触发条件 | 通知方式 |
|----------|----------|----------|
| 支付成功 | 支付处理成功 | WebSocket + 可选邮件/短信 |
| 支付失败 | 支付处理失败 | WebSocket + 错误提示 |
| 支付取消 | 用户主动取消 | WebSocket + 订单更新 |

### 7.2 异步处理流程

```java
/**
 * 异步支付处理流程
 */
public CompletableFuture<PaymentResult> processPaymentAsync(Long orderId, Long userId) {
    // 1. 创建CompletableFuture
    CompletableFuture<PaymentResult> future = new CompletableFuture<>();
    
    // 2. 提交到线程池处理
    CompletableFuture.supplyAsync(() -> {
        try {
            // 处理支付逻辑
            return doProcessPayment(orderId, userId);
        } catch (Exception e) {
            future.completeExceptionally(e);
            return null;
        }
    }, paymentTaskExecutor).thenAccept(result -> {
        // 3. 处理完成后通知
        future.complete(result);
    });
    
    // 4. 立即返回Future对象
    return future;
}
```

## 8. 使用指南

### 8.1 发起支付

```java
// 准备支付请求
PaymentRequest request = new PaymentRequest();
request.setOrderId(1L);
request.setAsync(true); // 使用异步处理

// 发起支付
CompletableFuture<PaymentResult> future = paymentService.processPayment(
    request.getOrderId(), 
    request.getUserId()
);

// 异步处理结果
future.thenAccept(result -> {
    if (result.isSuccess()) {
        System.out.println("支付成功: " + result.getAmount());
    } else {
        System.out.println("支付失败: " + result.getMessage());
    }
});
```

## 10. 常见问题

### Q1: 模拟支付的失败率可以调整吗？

**A**: 可以，在 `PaymentService` 中修改 `simulatePaymentProcessing` 方法的失败概率。

### Q2: 如何接入真实支付渠道？

**A**: 创建新的支付实现类，实现统一的支付接口，替换模拟支付逻辑。

### Q3: 异步支付超时如何处理？

**A**: 设置合理的超时时间，超时后通过查询接口确认支付状态。

### Q4: 支付失败后如何重试？

**A**: 客户端捕获支付失败通知后，可以重新发起支付请求。

### Q5: 如何确保支付结果的准确性？

**A**: 使用数据库事务保证支付状态的一致性，并通过状态查询接口进行确认。
