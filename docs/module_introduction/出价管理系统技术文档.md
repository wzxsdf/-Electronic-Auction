## 1. 系统概述

### 1.1 功能简介

出价管理模块负责处理用户在拍卖过程中的出价行为，是拍卖系统最核心的实时交互模块，包括：

- ✅ **实时出价**: 处理用户的实时出价请求
- ✅ **价格验证**: 验证出价的合法性和有效性
- ✅ **排名计算**: 实时计算用户的出价排名
- ✅ **领先通知**: 通知用户当前的领先状态
- ✅ **超越提醒**: 当用户被超越时发送提醒
- ✅ **频率控制**: 防止恶意频繁出价
- ✅ **排行榜管理**: 维护实时的出价排行榜

### 1.2 设计目标

- **实时性**: 毫秒级的出价处理响应
- **准确性**: 保证出价数据的绝对准确
- **并发安全**: 支持高并发出价场景
- **公平性**: 确保出价顺序的公平性
- **可扩展性**: 支持自动出价等扩展功能

### 1.3 技术特点

| 特性 | 技术方案 | 优势 |
|------|----------|------|
| 并发控制 | Redis分布式锁 | 解决并发冲突 |
| 实时推送 | WebSocket | 即时状态更新 |
| 缓存策略 | Redis缓存 | 提升查询性能 |
| 排行榜 | Redis Sorted Set | 高效排名计算 |
| 限流保护 | Redis计数器 | 防止频繁操作 |

## 4. 核心功能

### 4.1 实时出价处理

```java
/**
 * 出价处理核心逻辑
 */
public BidResultResponse placeBid(PlaceBidRequest request) {
    // 1. 获取分布式锁
    String lockKey = DistributedLockService.bidLockKey(request.getAuctionId());
    
    return distributedLockService.executeWithLock(lockKey, () -> {
        // 2. 在事务内执行出价逻辑
        return placeBidInternal(request);
    });
}

/**
 * 内部出价逻辑（在分布式锁保护下执行）
 */
@Transactional(rollbackFor = Exception.class)
public BidResultResponse placeBidInternal(PlaceBidRequest request) {
    // 验证拍卖和用户
    Auction auction = auctionRepository.findById(request.getAuctionId());
    User user = userRepository.findById(request.getUserId());
    
    // 校验出价合法性
    validateBid(auction, request.getUserId(), request.getAmount());
    
    // 更新拍卖状态
    updateAuctionState(auction, request.getAmount(), request.getUserId());
    
    // 更新排行榜
    updateLeaderboard(auction.getId(), request.getUserId(), request.getAmount());
    
    // 保存出价记录
    Bid bid = saveBidRecord(request);
    
    // 计算排名
    Integer rank = calculateRank(auction.getId(), request.getUserId());
    
    // 发送实时通知
    sendBidNotifications(auction, bid, user, rank);
    
    // 返回出价结果
    return buildBidResult(bid, rank, auction);
}
```

### 4.2 价格验证机制

```java
private void validateBid(Auction auction, Long userId, BigDecimal amount) {
    // 1. 验证拍卖状态
    if (auction.getStatusEnum() != AuctionStatus.ACTIVE) {
        throw new BizException(ErrorCode.AUCTION_NOT_ACTIVE);
    }
    
    // 2. 验证出价金额
    BigDecimal currentPrice = getCurrentPrice(auction);
    if (amount.compareTo(currentPrice) <= 0) {
        throw new BizException(ErrorCode.BID_AMOUNT_TOO_LOW);
    }
    
    // 3. 验证加价幅度
    BigDecimal minIncrement = auction.getBidIncrement();
    BigDecimal minValidPrice = currentPrice.add(minIncrement);
    if (amount.compareTo(minValidPrice) < 0) {
        throw new BizException(ErrorCode.BID_AMOUNT_INVALID);
    }
    
    // 4. 验证封顶价
    if (auction.getMaxPrice() != null && amount.compareTo(auction.getMaxPrice()) > 0) {
        throw new BizException(ErrorCode.BID_EXCEED_MAX_PRICE);
    }
    
    // 5. 验证是否为当前领先者
    Long highestBidder = getHighestBidder(auction);
    if (userId.equals(highestBidder)) {
        throw new BizException(ErrorCode.BID_ALREADY_LEADING);
    }
    
    // 6. 检查出价频率
    checkBidFrequency(userId);
}
```

### 4.3 排行榜管理

```java
/**
 * 更新排行榜
 */
private void updateLeaderboard(Long auctionId, Long userId, BigDecimal amount) {
    String leaderboardKey = "auction:" + auctionId + ":leaderboard";
    // 使用Redis Sorted Set维护排行榜
    redisService.zAdd(leaderboardKey, userId.toString(), amount.doubleValue());
}

/**
 * 计算用户排名
 */
private Integer calculateRank(Long auctionId, Long userId) {
    String leaderboardKey = "auction:" + auctionId + ":leaderboard";
    // 获取用户在排行榜中的排名（从0开始，加1转换为从1开始）
    Long rank = redisService.zReverseRank(leaderboardKey, userId.toString());
    return rank != null ? rank.intValue() + 1 : null;
}

/**
 * 获取排行榜Top N
 */
public List<LeaderboardEntry> getTopBidders(Long auctionId, int limit) {
    String leaderboardKey = "auction:" + auctionId + ":leaderboard";
    Set<String> topUsers = redisService.zReverseRange(leaderboardKey, 0, limit - 1);
    
    return topUsers.stream()
        .map(userId -> {
            Double score = redisService.zScore(leaderboardKey, userId);
            User user = userRepository.findById(Long.parseLong(userId));
            return new LeaderboardEntry(user, score);
        })
        .collect(Collectors.toList());
}
```

### 4.4 实时通知推送

```java
/**
 * 发送出价相关通知
 */
private void sendBidNotifications(Auction auction, Bid bid, User user, Integer rank) {
    // 1. 广播最新出价信息
    wsMessageService.broadcastNewBid(
        auction, 
        bid, 
        user.getNickname() != null ? user.getNickname() : "用户***",
        rank
    );
    
    // 2. 广播价格更新
    wsMessageService.broadcastPriceUpdate(auction.getId());
    
    // 3. 通知被超越的用户
    Long previousHighestBidder = getPreviousHighestBidder(auction.getId());
    if (previousHighestBidder != null && !previousHighestBidder.equals(bid.getUserId())) {
        wsMessageService.sendYouWereOvertaken(
            previousHighestBidder, 
            auction.getId(), 
            bid.getAmount()
        );
    }
    
    // 4. 通知当前领先者
    wsMessageService.sendYouAreLeading(
        bid.getUserId(), 
        auction.getId(), 
        bid.getAmount()
    );
}
```

## 6. 业务规则

### 6.1 出价验证规则

| 规则类型 | 验证条件 | 错误提示 |
|----------|----------|----------|
| 拍卖状态 | 拍卖必须为ACTIVE状态 | 拍卖未开始或已结束 |
| 价格递增 | 出价必须高于当前价格 | 出价金额过低 |
| 加价幅度 | 出价必须按加价幅度递增 | 出价不符合幅度要求 |
| 封顶价限制 | 出价不能超过封顶价 | 出价超过封顶价 |
| 领先限制 | 当前领先者不能连续出价 | 您当前是最高出价者 |
| 频率限制 | 1分钟内最多30次出价 | 出价频率过高 |

### 6.2 排行榜规则

- **实时更新**: 每次出价后立即更新排行榜
- **排名方式**: 按出价金额从高到低排名
- **相同价格**: 先出价者排名靠前
- **脱敏显示**: 排行榜中的用户名脱敏显示

### 6.3 通知规则

| 通知类型 | 触发条件 | 接收者 |
|----------|----------|--------|
| 出价成功 | 每次有效出价 | 出价者 |
| 被超越 | 原领先者被超越 | 原领先者 |
| 当前领先 | 成为最高出价者 | 当前领先者 |
| 价格更新 | 每次出价后 | 所有参与者 |

## 7. 并发控制

### 7.1 分布式锁机制

```java
/**
 * 使用Redis分布式锁保证并发安全
 */
public <T> T executeWithLock(String lockKey, Supplier<T> action) {
    // 1. 尝试获取锁
    boolean locked = redisService.setNx(lockKey, "1", 10, TimeUnit.SECONDS);
    
    if (!locked) {
        throw new BizException(ErrorCode.SYSTEM_BUSY, "系统繁忙，请稍后再试");
    }
    
    try {
        // 2. 执行业务逻辑
        return action.get();
    } finally {
        // 3. 释放锁
        redisService.del(lockKey);
    }
}
```

```java
/**
 * 确保Redis缓存与MySQL数据一致
 */
@Transactional(rollbackFor = Exception.class)
public void updateAuctionState(Auction auction, BigDecimal newPrice, Long userId) {
    // 1. 先更新MySQL
    auction.setCurrentPrice(newPrice);
    auction.setHighestBidder(userId);
    auctionRepository.updateById(auction);
    
    // 2. 再更新Redis缓存
    String auctionKey = "auction:" + auction.getId();
    redisService.set(auctionKey + ":current_price", newPrice.toString());
    redisService.set(auctionKey + ":highest_bidder", userId.toString());
}
```

```java
@Test
public void testConcurrentBids() throws InterruptedException {
    int threadCount = 100;
    CountDownLatch latch = new CountDownLatch(threadCount);
    
    for (int i = 0; i < threadCount; i++) {
        final int userId = i;
        new Thread(() -> {
            try {
                PlaceBidRequest request = new PlaceBidRequest();
                request.setAuctionId(1L);
                request.setUserId((long) userId);
                request.setAmount(new BigDecimal(1000 + userId * 50));
                
                bidService.placeBid(request);
            } finally {
                latch.countDown();
            }
        }).start();
    }
    
    latch.await();
    
    // 验证最终状态
    Auction auction = auctionRepository.findById(1L);
    assertEquals(100 + 99 * 50, auction.getCurrentPrice().intValue());
}
```

## 10. 常见问题

### Q1: 如何处理出价冲突？

**A**: 使用Redis分布式锁确保同一时刻只有一个出价请求能被处理，其他请求等待或重试。

### Q2: 出价频率限制如何实现？

**A**: 使用Redis计数器，每用户1分钟内最多30次出价，超过则拒绝请求。

### Q3: 如何保证排行榜的准确性？

**A**: 使用Redis Sorted Set维护排行榜，每次出价后更新，确保数据准确性。

### Q4: 被超越通知如何保证及时性？

**A**: 使用WebSocket实时推送，出价成功后立即通知被超越的用户。

### Q5: 如何防止恶意出价？

**A**: 通过频率限制、金额验证、分布式锁等多种机制防止恶意出价行为。
