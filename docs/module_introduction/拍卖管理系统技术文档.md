## 1. 系统概述

### 1.1 功能简介

拍卖管理模块是电子拍卖系统的核心业务模块，负责拍卖活动的完整生命周期管理，包括：

- ✅ **拍卖创建**: 商家创建拍卖活动，设置拍卖规则
- ✅ **状态管理**: 拍卖状态的完整流转控制
- ✅ **实时更新**: 拍卖进度的实时跟踪和更新
- ✅ **自动结算**: 拍卖结束后的自动结算处理
- ✅ **异常处理**: 拍卖异常情况的检测和处理
- ✅ **通知推送**: 拍卖状态变更的实时通知

### 1.2 设计目标

- **可靠性**: 保证拍卖状态的一致性和准确性
- **实时性**: 提供实时的拍卖状态更新
- **自动化**: 自动处理拍卖结束和结算
- **安全性**: 防止并发操作导致的数据不一致
- **扩展性**: 支持多种拍卖模式和规则配置

### 1.3 技术特点

| 特性 | 技术方案 | 优势 |
|------|----------|------|
| 并发控制 | 分布式锁 + 乐观锁 | 防止并发操作冲突 |
| 实时通信 | WebSocket | 低延迟的状态推送 |
| 缓存加速 | Redis缓存 | 提升查询性能 |
| 延时拍卖 | 动态时间调整 | 防止最后时刻狙击 |
| 状态机 | 严格状态流转 | 保证业务逻辑正确性 |

## 3. 数据库设计

### 3.1 拍卖表 (auctions)

```sql
CREATE TABLE `auctions` (
  `id`                BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '拍卖ID',
  `product_id`        BIGINT NOT NULL COMMENT '商品ID',
  `title`             VARCHAR(200) NOT NULL COMMENT '拍卖标题',
  `start_price`       DECIMAL(10,2) NOT NULL COMMENT '起拍价',
  `current_price`     DECIMAL(10,2) NOT NULL COMMENT '当前价格',
  `bid_increment`     DECIMAL(10,2) NOT NULL COMMENT '加价幅度',
  `max_price`         DECIMAL(10,2) COMMENT '封顶价',
  `delay_seconds`     INT DEFAULT 15 COMMENT '自动延时时长（秒）',
  `start_time`         DATETIME NOT NULL COMMENT '开始时间',
  `end_time`           DATETIME NOT NULL COMMENT '结束时间',
  `original_end_time`  DATETIME NOT NULL COMMENT '原始结束时间',
  `highest_bidder`     BIGINT COMMENT '当前最高出价者ID',
  `bid_count`          INT DEFAULT 0 COMMENT '出价次数',
  `status`             VARCHAR(20) NOT NULL DEFAULT 'PENDING' COMMENT '拍卖状态',
  `created_at`         DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at`         DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `version`            INT DEFAULT 0 COMMENT '乐观锁版本号',
  INDEX `idx_product_id` (`product_id`),
  INDEX `idx_status` (`status`),
  INDEX `idx_start_time` (`start_time`),
  INDEX `idx_end_time` (`end_time`),
  FOREIGN KEY (`product_id`) REFERENCES `products`(`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='拍卖表';
```

**字段说明**:
- `id`: 主键，自增长
- `status`: 拍卖状态（PENDING, ACTIVE, COMPLETED, CANCELLED）
- `current_price`: 当前最高出价，实时更新
- `highest_bidder`: 当前最高出价者ID
- `end_time`: 当前结束时间，可能因延时而变化
- `original_end_time`: 原始结束时间，固定不变
- `version`: 乐观锁版本号，防止并发更新冲突

## 4. 核心功能

### 4.1 拍卖创建流程

```
┌─────────────────────────────────────────────────────────────┐
│                      拍卖创建流程                             │
└─────────────────────────────────────────────────────────────┘
 1. 商家提交拍卖信息
    │
    ├─ 选择商品
    ├─ 设置起拍价
    ├─ 设置加价幅度
    ├─ 设置封顶价（可选）
    ├─ 设置拍卖时间
    └─ 设置延时规则
    │
    ▼
 2. 参数校验
    │
    ├─ 商品是否存在
    ├─ 价格是否合法
    ├─ 时间是否合理
    └─ 规则是否完整
    │
    ▼
 3. 创建拍卖记录
    │
    ├─ 状态设为PENDING
    ├─ 初始化当前价格为起拍价
    └─ 设置结束时间
    │
    ▼
 4. 缓存拍卖信息
    │
    ├─ Redis缓存基本信息
    └─ 初始化排行榜
    │
    ▼
 5. 返回创建结果
```

### 4.2 拍卖状态流转

```
┌─────────────────────────────────────────────────────────────┐
│                    拍卖状态流转图                             │
└─────────────────────────────────────────────────────────────┘

 PENDING (待开始)
    │
    ├─ 时间到达 → ACTIVE (进行中)
    │
    └─ 管理员取消 → CANCELLED (已取消)

 ACTIVE (进行中)
    │
    ├─ 到达结束时间 → COMPLETED (已完成)
    ├─ 达到封顶价 → COMPLETED (已完成)
    │
    └─ 异常情况 → CANCELLED (已取消)

 COMPLETED (已完成)
    │
    └─ 自动结算 → 生成订单

 CANCELLED (已取消)
    │
    └─ 通知所有参与者
```

### 4.3 自动延时机制

```
┌─────────────────────────────────────────────────────────────┐
│                    自动延时处理流程                          │
└─────────────────────────────────────────────────────────────┘
 1. 监控出价时间
    │
    ├─ 记录每次出价时间
    └─ 计算距离结束时间
    │
    ▼
 2. 判断延时条件
    │
    ├─ IF (剩余时间 < 延时阈值)
    │   AND (有新出价)
    │   AND (未达到封顶价)
    │   THEN 触发延时
    │
    ▼
 3. 执行延时操作
    │
    ├─ 延长结束时间
    ├─ 更新Redis缓存
    ├─ 发送延时通知
    └─ 记录延时日志
    │
    ▼
 4. 继续监控
```

## 6. 业务规则

### 6.1 拍卖创建规则

- **时间规则**: 开始时间必须早于结束时间
- **价格规则**: 起拍价必须大于0，加价幅度必须大于0
- **封顶价规则**: 封顶价必须大于起拍价
- **商品规则**: 商品必须存在且可用
- **权限规则**: 只有商家和主播可以创建拍卖

### 6.2 出价规则

- **价格递增**: 出价必须高于当前价格
- **加价幅度**: 出价必须按设定的加价幅度递增
- **封顶价限制**: 出价不能超过封顶价
- **领先限制**: 当前最高出价者不能连续出价
- **频率限制**: 单用户每分钟最多30次出价

### 6.3 延时规则

- **触发条件**: 剩余时间<延时阈值时，有新出价则触发延时
- **延时长度**: 默认15秒，可配置
- **最大延时**: 通常设置最大延时次数防止无限延长
- **封顶价例外**: 达到封顶价立即结束，不触发延时

### 6.4 结算规则

- **结束条件**: 时间到期 OR 达到封顶价 OR 主动取消
- **成交条件**: 至少有一次有效出价
- **订单生成**: 成交后自动为最高出价者生成订单
- **通知机制**: 结束后通知所有参与者

### 7.3 处理拍卖结束

```java
// 拍卖结束时的处理
public void handleAuctionEnd(Long auctionId) {
    Auction auction = auctionRepository.findById(auctionId);
    
    if (auction.getHighestBidder() != null) {
        // 有最高出价者，生成订单
        Order order = new Order();
        order.setUserId(auction.getHighestBidder());
        order.setItemId(auctionId);
        order.setProductId(auction.getProductId());
        order.setFinalAmount(auction.getCurrentPrice());
        orderService.createOrder(order);
        
        // 发送成交通知
        wsMessageService.sendAuctionCompleted(auctionId);
    } else {
        // 无人出价，发送流拍通知
        wsMessageService.sendAuctionFailed(auctionId);
    }
}
```

## 9. 常见问题

### Q1: 如何修改拍卖的延时规则？

**A**: 在创建拍卖时设置 `delaySeconds` 参数，或在拍卖开始前修改拍卖配置。

### Q2: 拍卖可以无限延时吗？

**A**: 不可以，可以设置最大延时次数限制，通常建议最多延时3-5次。

### Q3: 封顶价如何起作用？

**A**: 当出价达到封顶价时，拍卖立即结束，该出价者以封顶价成交。

### Q4: 如何处理拍卖异常？

**A**: 可以通过取消接口处理异常情况，系统会通知所有参与者并清理相关数据。
