## 1. 系统概述

### 1.1 功能简介

用户认证和权限模块为拍卖系统提供完整的用户管理和权限控制功能，包括：

- ✅ **用户注册与登录**: 支持用户名密码注册登录
- ✅ **JWT认证**: 基于JWT的无状态认证机制
- ✅ **权限管理**: RBAC角色权限模型
- ✅ **用户管理**: 用户的增删改查、禁用启用
- ✅ **安全防护**: 密码加密、防暴力破解、限流保护
- ✅ **日志审计**: 登录日志和操作日志记录

### 1.2 设计目标

- **安全性**: 生产级别的安全标准，保护用户数据和系统安全
- **可靠性**: 高可用性设计，支持高并发访问
- **可扩展性**: 模块化设计，易于扩展新功能
- **易用性**: 清晰的API设计，便于集成和使用
- **兼容性**: 与现有拍卖系统架构完美融合

### 1.3 技术特点

| 特性 | 技术方案 | 优势 |
|------|----------|------|
| 认证方式 | JWT + Redis | 无状态、高性能、易扩展 |
| 密码加密 | BCrypt | 单向加密、自适应复杂度、内置盐值 |
| 权限模型 | RBAC | 简洁清晰、易于理解和维护 |
| 数据访问 | MyBatis-Plus | 强大的CRUD操作、支持复杂查询 |
| 接口限流 | Redis + Lua | 分布式限流、高性能、原子性 |
| 日志记录 | 异步处理 | 不影响主流程性能 |

### 2.2 分层设计

#### 控制器层 (Controller)
- **职责**: 处理HTTP请求，参数校验，调用业务逻辑
- **特点**: 统一的Result响应格式，异常处理，参数验证
- **主要类**: `AuthController`, `UserController`, `RoleController`

#### 业务逻辑层 (Service)
- **职责**: 实现核心业务逻辑，事务管理，权限验证
- **特点**: 可重用，可测试，职责单一
- **主要类**: `AuthService`, `UserService`, `JwtService`, `LogService`

#### 数据访问层 (Repository)
- **职责**: 数据库CRUD操作，数据查询
- **特点**: 基于MyBatis-Plus，支持复杂查询
- **主要类**: `UserRepository`, `RoleRepository`, `PermissionRepository`

### 3.2 用户表 (users)

```sql
CREATE TABLE `users` (
  `id`              BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
  `username`        VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
  `password`        VARCHAR(100) NOT NULL COMMENT '密码（BCrypt加密）',
  `nickname`        VARCHAR(100) COMMENT '昵称',
  `avatar_url`      VARCHAR(500) COMMENT '头像URL',
  `email`           VARCHAR(100) UNIQUE COMMENT '邮箱',
  `phone`           VARCHAR(20) UNIQUE COMMENT '手机号',
  `status`          VARCHAR(20) NOT NULL DEFAULT 'ACTIVE' COMMENT '用户状态',
  `last_login_at`   DATETIME COMMENT '最后登录时间',
  `last_login_ip`   VARCHAR(50) COMMENT '最后登录IP',
  `total_bids`      INT DEFAULT 0 COMMENT '总出价次数',
  `total_wins`      INT DEFAULT 0 COMMENT '总获胜次数',
  `created_at`      DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at`      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `version`         INT DEFAULT 0 COMMENT '乐观锁版本号',
  INDEX `idx_username` (`username`),
  INDEX `idx_email` (`email`),
  INDEX `idx_phone` (`phone`),
  INDEX `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**字段说明**:
- `id`: 主键，自增长
- `username`: 用户名，唯一，4-20位字母数字下划线
- `password`: BCrypt加密后的密码
- `status`: 用户状态（ACTIVE, DISABLED, LOCKED, PENDING_VERIFICATION, DELETED）
- `version`: 乐观锁版本号，防止并发更新冲突

## 4. 核心功能

### 4.1 用户注册流程

```
┌─────────────────────────────────────────────────────────────┐
│                      用户注册流程                             │
└─────────────────────────────────────────────────────────────┘
 1. 客户端提交注册信息
    │
    ├─ 用户名（4-20位字母数字下划线）
    ├─ 密码（8-20位大小写字母+数字）
    ├─ 昵称（2-20位）
    ├─ 邮箱（可选，格式验证）
    └─ 手机号（可选，格式验证）
    │
    ▼
 2. 参数校验
    │
    ├─ 用户名格式检查
    ├─ 密码强度检查
    ├─ 邮箱/手机号格式检查
    └─ 密码不能包含用户名
    │
    ▼
 3. 唯一性检查
    │
    ├─ 用户名是否存在
    ├─ 邮箱是否存在
    └─ 手机号是否存在
    │
    ▼
 4. 密码加密（BCrypt）
    │
    ▼
 5. 创建用户记录
    │
    ├─ 保存用户信息
    ├─ 状态设为ACTIVE
    └─ 分配默认角色（USER）
    │
    ▼
 6. 返回用户信息（不含密码）
```

### 4.2 用户登录流程

```
┌─────────────────────────────────────────────────────────────┐
│                      用户登录流程                             │
└─────────────────────────────────────────────────────────────┘
 1. 客户端提交登录信息
    │
    ├─ 用户名
    └─ 密码
    │
    ▼
 2. 检查登录锁
    │
    ├─ 检查用户名是否被锁定
    └─ 检查IP是否被锁定
    │
    ▼
 3. 查询用户信息
    │
    ▼
 4. 验证用户状态
    │
    ├─ 用户是否存在
    ├─ 用户是否被禁用
    ├─ 用户是否被锁定
    └─ 用户是否已注销
    │
    ▼
 5. 验证密码
    │
    ├─ BCrypt密码比对
    └─ 验证失败记录并增加计数
    │
    ▼
 6. 清除失败计数
    │
    ▼
 7. 生成Token
    │
    ├─ 生成Access Token（2小时）
    ├─ 生成Refresh Token（7天）
    └─ 缓存Refresh Token到Redis
    │
    ▼
 8. 记录登录日志
    │
    ├─ 记录成功状态
    ├─ 记录IP地址
    └─ 记录浏览器UA
    │
    ▼
 9. 更新用户登录信息
    │
    ├─ 更新最后登录时间
    └─ 更新最后登录IP
    │
    ▼
 10. 返回Token和用户信息
```

### 4.3 Token验证流程

```
┌─────────────────────────────────────────────────────────────┐
│                    JWT Token验证流程                          │
└─────────────────────────────────────────────────────────────┘
 1. 客户端请求携带Token
    │
    └─ Authorization: Bearer {access_token}
    │
    ▼
 2. JWT过滤器拦截
    │
    ├─ 提取Token
    ├─ 验证Token签名
    ├─ 检查Token过期时间
    └─ 检查Token类型
    │
    ▼
 3. 黑名单检查
    │
    └─ 检查Token是否已注销（Redis黑名单）
    │
    ▼
 4. 解析用户信息
    │
    ├─ 用户ID
    ├─ 用户名
    └─ 用户角色
    │
    ▼
 5. 设置请求上下文
    │
    ├─ request.setAttribute("userId", userId)
    ├─ request.setAttribute("username", username)
    └─ request.setAttribute("token", token)
    │
    ▼
 6. 继续处理请求
```

### 4.4 权限验证流程

```
┌─────────────────────────────────────────────────────────────┐
│                      权限验证流程                             │
└─────────────────────────────────────────────────────────────┘
 1. 请求到达Controller
    │
    ▼
 2. 提取用户信息
    │
    ├─ 从请求上下文获取userId
    └─ 从请求上下文获取用户角色
    │
    ▼
 3. 权限检查
    │
    ├─ 检查是否为资源所有者
    │   └─ 本人可以操作自己的资源
    │
    ├─ 检查是否为管理员
    │   └─ 管理员拥有所有权限
    │
    └─ 检查是否拥有所需权限
    │   ├─ 查询用户角色
    │   ├─ 查询角色权限
    │   └─ 验证权限匹配
    │
    ▼
 4. 权限验证结果
    │
    ├─ 验证通过 → 继续处理请求
    └─ 验证失败 → 返回403 Forbidden
```

### 5.2 认证接口

#### 5.2.1 用户注册

**接口地址**: `POST /auth/register`

**请求参数**:
```json
{
  "username": "testuser",
  "password": "Test123456",
  "nickname": "测试用户",
  "email": "test@example.com",
  "phone": "13800138000"
}
```

**参数校验**:
- `username`: 必填，4-20位字母、数字或下划线
- `password`: 必填，8-20位，包含大小写字母和数字
- `nickname`: 必填，2-20位
- `email`: 可选，有效的邮箱格式
- `phone`: 可选，有效的手机号格式

**响应示例**:
```json
{
  "code": 200,
  "message": "操作成功",
```

## 6. 安全机制

### 6.1 密码安全

#### 6.1.1 密码策略

**密码复杂度要求**:
- 长度：8-20位
- 必须包含：大写字母、小写字母、数字
- 可选字符：!@#$%*?&
- 不能包含用户名
- 不能与最近3次密码相同

**示例**:
```
✅ 正确密码：
- Test123456
- MyPassword@2026
- SecurePass123

❌ 错误密码：
- test123        (缺少大写字母)
- TEST123        (缺少小写字母)
- Test123        (少于8位)
- Testuser123    (包含用户名)
```

#### 6.1.2 密码加密

**加密算法**: BCrypt

**加密特性**:
- 单向加密，无法解密
- 自适应复杂度（随硬件性能提升）
- 内置盐值，防止彩虹表攻击
- 相同密码每次加密结果不同

**代码示例**:
```java
// 加密密码
String hashedPassword = BCryptPasswordEncoder().encode("Test123456");

// 验证密码
boolean matches = BCryptPasswordEncoder().matches("Test123456", hashedPassword);
```

#### 6.1.3 密码传输

**传输方式**: HTTPS

**安全措施**:
- 密码不在日志中记录
- 密码不在错误消息中显示
- 密码字段在响应中自动过滤
- 前端密码输入框使用 `type="password"`

### 6.2 Token安全

#### 6.2.1 Token结构

**JWT Token组成**:
```
Header.Payload.Signature

示例：
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoidGVzdHVzZXIiLCJyb2xlcyI6WyJVU0VSIl0sInR5cGUiOiJhY2Nlc3MiLCJleHAiOjE3MTY5NzgwMDB9.signature
```

**Payload包含信息**:
```json
{
  "userId": 1,
  "username": "testuser",
  "roles": ["USER"],
  "type": "access",
  "iat": 1716964200,
  "exp": 1716971400
}
```

#### 6.2.2 Token过期机制

| Token类型 | 有效期 | 用途 |
|-----------|--------|------|
| Access Token | 2小时 | API访问认证 |
| Refresh Token | 7天 | 刷新Access Token |

**刷新策略**:
- Access Token即将过期（剩余<30分钟）时自动刷新
- Refresh Token只能使用一次
- Refresh Token刷新后生成新的Access Token和Refresh Token

#### 6.2.3 Token黑名单

**黑名单机制**:
- 用户登出时，将Refresh Token加入Redis黑名单
- JWT过滤器验证Token时检查黑名单
- 黑名单Token有效期与Refresh Token一致

**Redis存储格式**:
```
key: token_blacklist:{userId}
value: {refresh_token}
ttl: 7天
```

### 6.3 防暴力破解

#### 6.3.1 登录失败限制

**限制策略**:
- 5次登录失败 → 锁定账户30分钟
- 同IP连续失败 → 锁定IP30分钟

**Redis存储格式**:
```
# 失败计数
login_fail_count:user:{username} = 3
login_fail_count:ip:{ip_address} = 5

# 锁定标记
login_lock:user:{username} = "locked"
login_lock:ip:{ip_address} = "locked"
```

#### 6.3.2 限流保护

**接口限流规则**:
| 接口 | 限流规则 | 持续时间 |
|------|----------|----------|
| 用户注册 | 5次 | 每小时 |
| 用户登录 | 10次 | 每分钟 |
| 修改密码 | 5次 | 每小时 |
| 用户查询 | 30次 | 每分钟 |

**限流算法**: Redis + Lua脚本实现的滑动窗口

### 6.4 数据安全

#### 6.4.1 敏感数据脱敏

**日志脱敏规则**:
```java
// 密码脱敏
"password":"***"

// Token脱敏
"token":"***"

// 身份证脱敏
"idCard":"***"

// 手机号脱敏（日志中）
138****8000
```

**API响应脱敏**:
```java
// 用户名脱敏（出价历史）
testuser -> tes***

// 手机号脱敏（普通用户查询）
13800138000 -> 138****8000
```

#### 6.4.2 SQL注入防护

**防护措施**:
- 使用MyBatis-Plus预编译SQL
- 禁止拼接SQL字符串
- 输入参数校验和过滤
- 使用参数化查询

**示例**:
```java
// 安全：使用参数化查询
LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(User::getUsername, username);
User user = userMapper.selectOne(wrapper);

// 不安全：字符串拼接（禁止）
String sql = "SELECT * FROM users WHERE username = '" + username + "'";
```

#### 6.4.3 XSS攻击防护

**防护措施**:
- 前端输入过滤和转义
- 后端输出HTML编码
- Content-Type设置为application/json
- 使用安全的模板引擎

### 6.5 权限控制

#### 6.5.1 权限模型

**RBAC权限模型**:
```
用户(User) → 角色(Role) → 权限(Permission) → 资源(Resource)

示例：
用户张三 → 角色USER → 权限auction:bid → 资源/api/bids
```

#### 6.5.2 权限验证

**验证层次**:
1. **认证层**: 验证用户身份（JWT Token）
2. **授权层**: 验证用户权限（角色权限）
3. **资源层**: 验证资源所有权（本人资源）

**验证流程**:
```
请求 → Token验证 → 角色权限查询 → 资源所有权检查 → 允许/拒绝
```

```javascript
        }
```

```python
        result = response.json()
        
        if result['code'] == 200:
            self.access_token = result['data']
```

```java
import org.springframework.web.client.RestTemplate;
import org.springframework.http.*;

public class AuthClient {
    private static final String BASE_URL = "http://localhost:8080/api";
    private RestTemplate restTemplate = new RestTemplate();
    private String accessToken;
    private String refreshToken;

    public ResponseEntity<?> register(String username, String password, String nickname) {
        String url = BASE_URL + "/auth/register";
        
        Map<String, Object> requestBody = new HashMap<>();
        requestBody.put("username", username);
        requestBody.put("password", password);
        requestBody.put("nickname", nickname);
        
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<Map<String, Object>> request = new HttpEntity<>(requestBody, headers);
        return restTemplate.postForEntity(url, request, Map.class);
    }

    public ResponseEntity<?> login(String username, String password) {
        String url = BASE_URL + "/auth/login";
        
        Map<String, String> requestBody = new HashMap<>();
        requestBody.put("username", username);
        requestBody.put("password", password);
        
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<Map<String, String>> request = new HttpEntity<>(requestBody, headers);
        ResponseEntity<Map> response = restTemplate.postForEntity(url, request, Map.class);
        
        if (response.getStatusCode() == HttpStatus.OK) {
            Map<String, Object> body = response.getBody();
            if (body != null && body.get("code").equals(200)) {
                Map<String, Object> data = (Map<String, Object>) body.get("data");
                accessToken = (String) data.get("accessToken");
                refreshToken = (String) data.get("refreshToken");
            }
        }
        
        return response;
    }

    public ResponseEntity<?> getProfile() {
        String url = BASE_URL + "/auth/me";
        
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + accessToken);
        
        HttpEntity<?> request = new HttpEntity<>(headers);
        return restTemplate.exchange(url, HttpMethod.GET, request, Map.class);
    }

    public static void main(String[] args) {
        AuthClient client = new AuthClient();
        
        // 注册
        ResponseEntity<?> registerResponse = client.register("testuser", "Test123456", "测试用户");
        System.out.println("注册结果: " + registerResponse.getBody());
```

### 10.2 技术相关

#### Q4: 如何处理Token过期问题？

**A**: 前端实现自动刷新机制：

```javascript
// 请求拦截器
axios.interceptors.request.use(async config => {
  const token = localStorage.getItem('accessToken');
  
  // 检查Token是否即将过期
  if (isTokenExpiringSoon(token)) {
    const newToken = await refreshAccessToken();
    localStorage.setItem('accessToken', newToken);
    config.headers.Authorization = `Bearer ${newToken}`;
  } else {
    config.headers.Authorization = `Bearer ${token}`;
  }
  
  return config;
});

// 响应拦截器
axios.interceptors.response.use(
  response => response,
  async error => {
    if (error.response.status === 401) {
      // Token过期，尝试刷新
      try {
        const newToken = await refreshAccessToken();
        localStorage.setItem('accessToken', newToken);
        // 重试原请求
        error.config.headers.Authorization = `Bearer ${newToken}`;
        return axios.request(error.config);
      } catch (refreshError) {
        // 刷新失败，跳转登录页
        localStorage.clear();
        window.location.href = '/login';
      }
    }
    return Promise.reject(error);
  }
);
```
