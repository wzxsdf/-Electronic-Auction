## 六、索引与外键优化

### 新增关键索引
```sql
-- users表
uk_username: 用户名唯一索引
idx_email: 邮箱索引
idx_phone: 手机号索引
idx_status: 状态索引

-- auction_items表
idx_product: 商品ID索引（用于查询商品的所有竞拍）

-- bids表
idx_item_amount: (item_id, amount DESC) 用于查询最高出价
```

### 外键约束
- 所有外键使用RESTRICT或CASCADE策略
- 确保数据引用完整性
- 级联删除合理配置（如user_roles、role_permissions）

## 十、新增表详细说明

### 1. roles表（角色表）
```sql
字段说明：
- code: ADMIN, USER等角色编码
- name: 角色显示名称
- description: 角色描述
```

### 2. permissions表（权限表）
```sql
字段说明：
- code: user:read, auction:write等权限编码
- name: 权限显示名称
- resource: 资源路径（如/api/users/**）
- action: 操作类型（READ, WRITE, DELETE）
```

### 3. login_logs表（登录日志）
```sql
字段说明：
- login_type: 登录类型（PASSWORD, SOCIAL, SSO）
- status: 登录状态（SUCCESS, FAILED）
- failure_reason: 失败原因
```

### 4. operation_logs表（操作日志）
```sql
字段说明：
- module: 操作模块（auction, order, user等）
- operation: 操作类型（CREATE, UPDATE, DELETE）
- method: 请求方法路径
- params: 请求参数
- duration: 执行耗时
```
